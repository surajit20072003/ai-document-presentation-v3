### MANIM_SCENE_SPEC WRITING GUIDE
A good manim_scene_spec describes:
- What Manim objects appear first and when (example: "At 0s, dark background. At 1s, a right triangle draws stroke-by-stroke using Polygon in TEAL, with vertices labeled A, B, C using Text in BLUE")
- Colours using Manim constants (GOLD, TEAL, RED, BLUE, WHITE, YELLOW, GREEN, PURPLE)
- LaTeX formulas to display (example: `MathTex(r"\sin\theta = \frac{\text{opposite}}{\text{hypotenuse}}")`)
- Label text exactly (quote the text that should appear)
- Animation sequence (Write, FadeIn, Create, Transform — timed to narration beats)
- What freezes at end (final static frame before FadeOut)

**⚠️ CRITICAL: EXPLICIT SECOND-LEVEL ANIMATION CHOREOGRAPHY REQUIRED**
The `manim_scene_spec` MUST include explicit second-level animation choreography.
Do NOT write vague specs like "show a triangle with labels".
Instead write exactly what appears, when, in what order.

GOOD EXAMPLE:
"At 0s: Create right triangle ABC using Polygon in TEAL over 2s. At 2s: Write label 'A' at vertex A, 'B' at vertex B, 'C' at vertex C in BLUE over 1s. At 3s: draw side BC highlighted in RED with Create over 2s. At 5s: draw hypotenuse AC in GOLD with GrowFromCenter over 3s. At 8s: FadeIn label 'Hypotenuse' near AC. At 10s: Write MathTex 'sin theta = opposite / hypotenuse' in GOLD at centre over 2s. Duration 15s. Complexity medium."

BAD EXAMPLE (will produce poor animation):
"Show a right triangle with sine, cosine and tangent labels. Duration 15s."

Rule: Every object, every label, every colour, every approximate timing — written explicitly.
The Manim code generator reads this spec and writes Python animation code from it directly.
Vague spec = vague animation.
