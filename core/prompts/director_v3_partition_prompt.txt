### IMAGE SOURCE HANDLING (C4 — When Image Detected in Source)
When the source document contains an image, `image_source` will be passed to you. Analyse the image and pick ONE of these three modes:

| image_mode | When to use | renderer |
|---|---|---|
| `video_reference` | Photograph / realistic illustration — best shown as cinematic video | `video` |
| `texture` | Diagram, chart, graph, table — information-dense, must stay readable | `threejs` |
| `interactive_hotspot` | Labelled diagram with parts needing individual explanation | `threejs` |

DECISION TREE:
1. Is the image a photograph or realistic illustration? → `video_reference`
2. Is the image a labelled diagram where each part needs individual explanation? → `interactive_hotspot`
3. Is the image a diagram, chart, or graph? → `texture`

When `image_mode` is set:
- Add `"image_mode": "<mode>"` and `"image_source": "<path>"` to the segment_spec
- For `interactive_hotspot`: interaction field MUST be non-null (use hover_highlight or click_reveal)
- For `video_reference`: no threejs_file, no threejs_spec — this beat is a video beat
- For `texture` or `interactive_hotspot`: threejs_spec must describe the image as a plane texture with animated labels/arrows on top

### THREEJS_SPEC WRITING GUIDE
A good threejs_spec describes:
- What geometry appears first and when (example: "At 0s, a dark background. At 1s, a right triangle draws stroke-by-stroke in teal, with vertices labeled A, B, C in sky blue")
- Colours (use palette names: gold=key highlight, teal=primary, rose=secondary, sky=labels)
- Label text exactly (quote the text that should appear on canvas)
- Animation sequence (progressive reveal timed to narration beats)
- What freezes at end (final static frame)
- Camera distance: 8 units default (change only for complex 3D scenes)

**⚠️ CRITICAL: EXPLICIT SECOND-LEVEL ANIMATION CHOREOGRAPHY REQUIRED**
The `threejs_spec` MUST include explicit second-level animation choreography.
Do NOT write vague specs like "show a triangle with labels".
Instead write exactly what appears, when, in what order.

GOOD EXAMPLE:
"At 0s: draw side AB in teal over 2s. At 2s: draw side BC in rose over 2s.
At 4s: draw hypotenuse AC in gold over 3s. At 7s: fade in label 'Hypotenuse' near AC.
At 9s: draw angle arc at A in lavender. At 11s: fade in formula
'sin θ = opposite / hypotenuse' in gold at centre. Duration 15s. Complexity medium."

BAD EXAMPLE (will produce poor animation):
"Show a right triangle with sine, cosine and tangent labels. Duration 15s."

Rule: Every object, every label, every colour, every approximate timing — written explicitly.
The code generator reads this spec and writes animation code from it directly.
Vague spec = vague animation.

=== INTERACTION RULES (MANDATORY) ===

Every segment_spec MUST include an "interaction" field.
Set it to null if the beat is animation-only (no student action needed).

FIVE ALLOWED TYPES:
  hover_highlight  — mouse over object → glows + tooltip. Any complexity.
  click_reveal     — click object → label/formula appears. Any complexity.
  drag_point       — drag vertex/point → geometry redraws live. complexity: medium or complex only.
  rotate_inspect   — drag anywhere → 3D object rotates. complexity: medium or complex only.
  slider           — drag handle on track → value changes (angle, speed, wavelength). complexity: complex only.

DECISION RULES:
  complexity: simple  → hover_highlight or click_reveal only. Set interaction: null if unsure.
  complexity: medium  → hover_highlight, click_reveal, drag_point, or rotate_inspect.
  complexity: complex → any single type. hover_highlight may be added as passive secondary.

NEVER set interaction (leave null) if:
  - segment_duration_seconds < 8.0
  - This is beat_0 (first beat of the section — student is still orienting)
  - The content is purely narrative with no manipulable scene object
  - More than one primary interaction type would be needed

NEVER combine drag_point + rotate_inspect in the same beat.
NEVER combine slider + drag_point in the same beat.

INTERACTION OBJECT SCHEMA (when non-null):
  {
    "type":            "<one of the 5 types above>",
    "target":          "<name of the interactive object, e.g. 'vertex_C', 'theta_slider'>",
    "description":     "<one sentence: what student does + what changes>",
    "timeout_seconds": 10
  }

WHAT TO WRITE IN threejs_spec FOR INTERACTIVITY:
  hover_highlight:  "Each side is hoverable. Hovering brightens the side to white and shows a gold tooltip label."
  click_reveal:     "The hypotenuse pulses to indicate it is clickable. Clicking brightens it to gold and fades in the 'Hypotenuse' label."
  drag_point:       "Vertex C is a draggable rose handle. Dragging it redraws BC and the angle arc at A in real time. Clamped to Y[0.5, 3.0]."
  rotate_inspect:   "Student drags to rotate the 3D prism freely. 'Drag to rotate' hint fades out after 3s."
  slider:           "Slider track at Y=-2.5. Gold handle controls theta 5°-85°. Triangle and sin/cos labels update in real time."
