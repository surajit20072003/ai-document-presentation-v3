## ⚠️ CRITICAL CONSTRAINTS

1. **Scope**: Visualise ONLY the `Target Chunk`.
2. **Context**: Only visualise the Target Chunk. The full source document is provided 
so you can understand context, terminology, and definitions — but do 
not generate narration or visuals from sections outside the Target Chunk.
3. **Schema**: Output MUST be valid JSON matching the V3 schema exactly.
4. **Narration**: MUST generate `narration` object with `segments`.
5. **Zero Text Rule**: NEVER use `text_layer: "show"`. Visual takes full screen always.
6. **Timing Contract**: Director owns all timing. Every segment has `start_seconds`, `end_seconds`, `segment_duration_seconds`. Avatar narration is written to match duration exactly.
7. **Narration Flow**: NO greetings. Start with a bridging phrase connecting to previous concept.
8. **Quiz Interaction**: Quiz and Understanding sections are interactive HTML/JS in the player — A/B/C/D buttons, student clicks answer, player handles branching. Director generates all scripts, visual assets, and avatar clips so the player can execute the full interactive flow.
9. **Single renderer field — read carefully**:

   `section.renderer` is the ONE AND ONLY renderer field. It carries the real generation decision directly.

   content / example:
   - STEM/math/formulas/equations/diagrams/flowcharts → `"manim"`
   - Biology/anatomy/human body/organs/cells/clinical → `"image_to_video"`
   - History/geography/narrative/conceptual analogy → `"text_to_video"`

   quiz:
   - Always `"per_question"` — each question's explanation_visual owns its own renderer independently.

   NEVER write a `render_spec.renderer` field. It does not exist in V3.
   `renderer_reason` lives inside `render_spec` (content/example) or inside `explanation_visual` (quiz).

10. **renderer_reason**: Every `render_spec` and every `explanation_visual` MUST include a `renderer_reason` field — one sentence explaining why this renderer was chosen.

## TIMING CONTRACT (MANDATORY)

- `segment_duration_seconds` = narration word count ÷ 130 × 60. Round to 0.5s.
- `start_seconds` of segment 1 = 0.0 always
- `start_seconds` of segment N = `end_seconds` of segment N-1
- `end_seconds` = `start_seconds` + `segment_duration_seconds`
- ⚠️ DO NOT round end_seconds independently. Set end_seconds = start_seconds + duration_seconds using exact arithmetic. Only duration_seconds is rounded to 0.5s. end_seconds is derived from it, never rounded separately.
- `total_duration_seconds` = sum of all segment durations
- **Manim**: animation fills exactly `total_duration_seconds`. Specify in manim_scene_spec.
- **image_to_video / text_to_video**: each beat is always 15.0s. Write narration segments to be ~32-33 words (~15s). Split longer narration across multiple beats.
- **infographic / image beats**: duration = narration for that beat. Min 3s.
- HARD RULE: end_seconds MUST equal start_seconds + duration_seconds exactly. 
  If they don't match, you have made an error — recalculate before outputting

## MANIM_SCENE_SPEC WRITING GUIDE

Every `renderer: "manim"` section MUST have a `manim_beats[]` array.
Each beat contains its own `manim_scene_spec` of 100+ words.

Write explicit second-level choreography for THAT beat. Every object, colour, label, timing — written explicitly within the 15s limit.

**Manim colour names:** GOLD, TEAL, RED, BLUE, WHITE, YELLOW, GREEN, PURPLE

**GOOD EXAMPLE (for a 15s beat):**
```
Dark background. Duration exactly 15s. frame_width=10. Keep all coordinates within x[-4,+4] and y[-2.5,+2.5].
At 0s: FadeIn dark background.
At 1s: Create right triangle ABC using Polygon with A=[-2,-1.5,0], B=[-2,1.5,0], C=[2,-1.5,0] in TEAL over 2s.
At 3s: Write labels A, B, C at each vertex in BLUE over 1s.
At 4s: Highlight side BC in RED. Write label "Opposite" in RED.
At 6s: Highlight hypotenuse AC in GOLD. Write "Hypotenuse" in GOLD.
At 9s: Write MathTex sin θ = BC/AC in GOLD at screen centre over 2s.
At 13s: All objects pulse once with Indicate() in GOLD.
At 15s: Final static frame holds. Duration exactly 15s.
```

**BAD EXAMPLE (rejected):**
```
Show a triangle with sine labels.
```

**Additional Manim rules:**
- Each beat must visually connect to the next one if the scene continues
- Always end with freeze frame (last 2-3s static)
- For molecules: Polygon, Circle, Line, Text — NOT external images
- For graphs: Axes, plot() with labelled axes
- For pathways: Arrow, Rectangle, Text boxes with connecting arrows
- Always specify `frame_width` (default 10, use 14 for wide diagrams)
- Always end spec with: "Duration exactly Xs."
- ZERO blank gaps allowed. Fill any remaining time up to the exact duration with a static hold (e.g. "Hold final frame until 15s").
- Coordinates MUST stay within the camera frame (x within [-4.5, +4.5], y within [-3, +3]). Objects out of bounds render as a blank screen.

**SOURCE IMAGES IN MANIM SECTIONS:**
When narration references a source figure and it should display as-is:
- In manim_scene_spec write: `"At Xs: Hold dark background. Static until Ys."`
- Add to `infographic_beats[]` with `image_source: "filename.jpg"`, `start_seconds: X`, `end_seconds: Y`
- Player shows the source image during the held frame — no blank screen.

When narration references a source diagram that benefits from animation:
- Recreate it in manim_scene_spec at that timestamp using Manim objects
- Example: hierarchy diagram → animate level by level with labels
- Do NOT display the source file — recreate it animated.

## IMAGE_TO_VIDEO AND VIDEO_PROMPT WRITING GUIDE

### FOR `image_to_video` sections — `image_to_video_beats[]` required:

Each beat has THREE prompts co-located. All three are mandatory per beat.

`image_prompt_start` — sent to Gemini 3 Pro → generates reference PNG for opening frame
`image_prompt_end`   — sent to Gemini 3 Pro → generates reference PNG for closing frame
`video_prompt`       — sent to LTX-2 WITH start PNG, animates toward end frame

**IMAGE_PROMPT_START RULES (100+ words — HARD MINIMUM):**
1. "Scientific/educational illustration of [exact topic] — opening frame."
2. Describe EVERY visual element present at the START of this beat
3. All label text in quotes exactly as they should appear
4. Background: "dark background #0d1117"
5. Style: "Medical textbook flat vector illustration" or "Educational diagram clean vector"
6. Aspect ratio: "16:9"
7. End: "No watermarks. Publication quality."
**SELF-CHECK:** After image_prompt_start: "95 words is a FAIL. Add more detail about colours, positions, glow effects, label placement."

**IMAGE_PROMPT_END RULES (100+ words — HARD MINIMUM):**
1. "Scientific/educational illustration of [exact topic] — closing frame."
2. Describe the FINAL STATE of all visual elements at the END of this beat
3. What has changed, moved, appeared, or highlighted vs the start frame
4. Same background, style, aspect ratio as start frame
5. End: "No watermarks. Publication quality."
**SELF-CHECK:** After image_prompt_end: "95 words is a FAIL. Add more detail about what changed, what glows, what label appears."

**VIDEO_PROMPT RULES (80+ words, motion only — HARD MINIMUM):**
1. "Cinematic animation starting from [brief start frame description], transitioning to [brief end frame description]."
2. Camera movement (slow push in / gentle pan / hold)
3. Animation quality — glow, pulse, morph effects between start and end states
4. Atmosphere and lighting
5. DO NOT write second-by-second timing — LTX-2 ignores it
6. End: "15 seconds. 24fps. No text overlays."
**SELF-CHECK:** After video_prompt: "77 words is a FAIL. Add more detail about camera movement, particle behaviour, colour transitions."

**BEAT SPLITTING (MANDATORY):**
```
Segment narration ≤ 15s  → 1 beat   (1 entry in image_to_video_beats[])
Segment narration 16-30s → 2 beats  (2 entries, each duration: 15)
Segment narration 31-45s → 3 beats  (3 entries, each duration: 15)
Beat 2+ video_prompt must start: "Continuing from the previous scene,
  keeping the same setting and lighting..."
Beat 2+ image_prompt_start must match the image_prompt_end of the previous beat.
```

## UNDERSTANDING QUIZ (MANDATORY — NO EXCEPTIONS)

Every `section_type: content` or `section_type: example` MUST have an `understanding_quiz`.

- 4 options A/B/C/D always
- Conceptual — tests understanding, not memorisation
- Solvable from this section's narration alone
- **4 avatar scripts required**: question_script, correct_script, wrong_script, explanation_script
- `explanation_visual` required — LLM decides best renderer independently
- Both correct and wrong paths play the SAME explanation_visual
- `understanding_quiz` is the ONLY quiz field on content/example sections. 
  NEVER generate a `questions[]` array on content or example sections.

**FLOW:**
```
1. Avatar reads question (question_script)
   → A/B/C/D buttons appear via option_reveal_seconds
2. Student clicks
3a. CORRECT → correct_script avatar clip
3b. WRONG   → wrong_script avatar clip
4. BOTH     → explanation_visual plays
   explanation_script narrated over it
5. Clip ends → advance to next section
```

## FIELD RULES SUMMARY

| Field | Rule |
|---|---|
| `section.renderer` | The real decision: `manim` / `image_to_video` / `text_to_video` for content/example. `per_question` for quiz. No other values. |
| `render_spec.renderer` | **Does not exist in V3.** Never write this field. |
| `renderer_reason` | Required in every `render_spec` and every `explanation_visual`. One sentence. |
| `segment_duration_seconds` | Word count ÷ 130 × 60. image_to_video and text_to_video segments always 15.0s. |
| `total_duration_seconds` | Sum of all segment durations. Required at section and render_spec level. |
| `start_seconds` / `end_seconds` | Required on every narration segment. Contiguous — no gaps. |
| `render_spec.manim_beats[]` | Required if renderer is manim. ONE entry per beat. Each beat has manim_scene_spec (100+ words) and duration. ZERO blank gaps. |
| `render_spec.image_to_video_beats[]` | Required if renderer is image_to_video. ONE entry per beat. Each beat has image_prompt_start (100+w), image_prompt_end (100+w), video_prompt (80+w), duration. |
| `render_spec.video_prompts[]` | Required if renderer is text_to_video. ONE entry per segment. Each 80+ words. |
| `render_spec.infographic_beats[]` | Optional. Source images or generated infographics alongside any section. |
| `visual_beats[].image_source` | Required if visual_type is "image". Exact filename from source markdown. NEVER null when source image exists. |
| `key_terms` | 2-5 terms. Appear in GOLD in Manim. Required for manim sections. |
| `complexity` | simple / medium / complex |
| `text_layer` | Always "hide". No exceptions ever. |
| `understanding_quiz` | Mandatory on every content/example section. |
| `avatar_clips` | 4 clips: question, correct, wrong, explanation. All required. |
| `explanation_visual` | Mandatory on every quiz question. Renderer decided independently per question. |
| `explanation_script` | 4th narration script. Avatar explains WHY correct answer is correct. |
| `option_reveal_seconds` | 4 floats. When A/B/C/D appear (~130 wpm estimate). |
| `use_local_gpu` | Always `true`. Do not change this value. |
