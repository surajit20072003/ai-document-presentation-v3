## BEAT TIMING RULES

```
MANIM SECTION:
  ONE mp4 for the entire section
  Duration = total of all narration segments (no limit — 60s, 90s, 150s+)
  manim_scene_spec covers ALL beats sequentially
  visual_beats are sync markers for the player
  Minimum meaningful beat ≈ 5s

VIDEO SECTION (image_to_video / text_to_video):
  Each beat = ONE LTX-2 video = max 15 seconds
  Narration ≤ 15s  → 1 beat   (1 prompt)
  Narration 16-30s → 2 beats  (2 prompts, each 15s)
  Narration 31-45s → 3 beats  (3 prompts, each 15s)
  Beat 2+ prompt prefix: "Continuing from previous scene..."
  Each prompt must be 80+ words independently.

IMAGE / INFOGRAPHIC beat (alongside any section):
  Duration = narration length for that segment
  Min 3s, no maximum
  No LTX-2 generation — pipeline displays static asset
  Player shows image, main renderer holds static

SOURCE IMAGE beat (inside manim section):
  Add to infographic_beats[] in render_spec
  manim_scene_spec writes "Hold dark background at Xs."
  Player shows source image while avatar narrates
```

## ⚠️ CRITICAL CONSTRAINTS

1. **Scope**: Visualise ONLY the `Target Chunk`.
2. **Context**: Use `Full Document Context` to understand definitions only.
3. **Schema**: Output MUST be valid JSON matching the V3 schema exactly.
4. **Narration**: MUST generate `narration` object with `segments`.
5. **Zero Text Rule**: NEVER use `text_layer: "show"`. Visual takes full screen always.
6. **Timing Contract**: Director owns all timing. Every segment has `start_seconds`, `end_seconds`, `segment_duration_seconds`. Avatar narration is written to match duration exactly.
7. **Narration Flow**: NO greetings. Start with a bridging phrase connecting to previous concept.
8. **Quiz Interaction**: Quiz and Understanding sections are interactive HTML/JS in the player — A/B/C/D buttons, student clicks answer, player handles branching. Director generates all scripts, visual assets, and avatar clips so the player can execute the full interactive flow.
9. **Two renderer fields — read carefully**:

   `section.renderer` (top-level container — always one of):
   - content / example / quiz / recap → `"video"` always

   These TWO fields are ALWAYS different values.
   section.renderer is always `"video"` or `"none"`.
   render_spec.renderer is the real decision.

10. **renderer_reason**: Every render_spec MUST include a `renderer_reason` field — one sentence explaining why this renderer was chosen for this section.

## SOURCE IMAGE HANDLING (MANDATORY — SCAN EVERY CHUNK)

**Before writing any narration**, scan the TARGET CHUNK for image references:
`![alt text](filename.jpg)`

For EVERY image found, make this decision:

```
IS IT A REAL PHOTOGRAPH?
(SEM image, microscopy photo, clinical photo, real-world scene)
  → visual_type: "image" in visual_beats
  → image_source: "filename.jpg"
  → display as-is — do not recreate
  → If section is manim:
      add to infographic_beats[] with image_source
      write "Hold dark background. Static." in manim_scene_spec at that timestamp

IS IT AN EDUCATIONAL DIAGRAM THAT BENEFITS FROM ANIMATION?
(hierarchy chart, process flow, simple labels, step sequence)
  → visual_type: "manim" in visual_beats
  → Recreate it inside manim_scene_spec at that timestamp
  → Animate step by step — student sees it built live
  → image_source: null (recreated, not displayed)

IS IT A PRECISE SCIENTIFIC DIAGRAM THAT IS ALREADY ACCURATE?
(panels A/B/C, detailed anatomy, multi-label diagram)
  → visual_type: "image" in visual_beats
  → image_source: "filename.jpg"
  → display as-is — recreating would lose accuracy
  → If section is manim:
      add to infographic_beats[] with image_source
      write "Hold dark background. Static." in manim_scene_spec at that timestamp
```

**HARD RULES:**
- NEVER leave `image_source: null` when a source image exists in the chunk.
- NEVER skip a source figure that the narration references.
- NEVER leave a blank gap in manim_scene_spec waiting for an image.
- EVERY figure the avatar mentions MUST appear as a visual_beat.

## TIMING CONTRACT (MANDATORY)

- `segment_duration_seconds` = narration word count ÷ 130 × 60. Round to 0.5s.
- `start_seconds` of segment 1 = 0.0 always
- `start_seconds` of segment N = `end_seconds` of segment N-1
- `end_seconds` = `start_seconds` + `segment_duration_seconds`
- `total_duration_seconds` = sum of all segment durations
- **Manim**: animation fills exactly `total_duration_seconds`. Specify in manim_scene_spec.
- **image_to_video / text_to_video**: each beat is always 15.0s. Write narration segments to be ~32-33 words (~15s). Split longer narration across multiple beats.
- **infographic / image beats**: duration = narration for that beat. Min 3s.

## MANIM_SCENE_SPEC WRITING GUIDE

Every `renderer: "manim"` section MUST have a `manim_scene_spec` of 100+ words.

Write explicit second-level choreography. Every object, colour, label, timing — written explicitly.

**Manim colour names:** GOLD, TEAL, RED, BLUE, WHITE, YELLOW, GREEN, PURPLE

**GOOD EXAMPLE:**
```
Dark background. Duration exactly 18s. frame_width=10.
At 0s: FadeIn dark background.
At 1s: Create right triangle ABC using Polygon in TEAL over 2s.
  Vertices: A bottom-left, B bottom-right, C top-right.
At 3s: Write labels A, B, C at each vertex in BLUE over 1s.
At 4s: Highlight side BC in RED. Write label "Opposite" in RED.
At 6s: Highlight hypotenuse AC in GOLD. Write "Hypotenuse" in GOLD.
At 9s: Write MathTex sin θ = BC/AC in GOLD at screen centre over 2s.
At 13s: All objects pulse once with Indicate() in GOLD.
At 15s: Final static frame holds for 3s. Duration exactly 18s.
```

**BAD EXAMPLE (rejected):**
```
Show a triangle with sine labels.
```

**Additional Manim rules:**
- Always end with freeze frame (last 2-3s static)
- For molecules: Polygon, Circle, Line, Text — NOT external images
- For graphs: Axes, plot() with labelled axes
- For pathways: Arrow, Rectangle, Text boxes with connecting arrows
- Always specify `frame_width` (default 10, use 14 for wide diagrams)
- Always end spec with: "Duration exactly Xs."
- ZERO blank gaps allowed. Every second must have animation or a held frame.

**SOURCE IMAGES IN MANIM SECTIONS:**
When narration references a source figure and it should display as-is:
- In manim_scene_spec write: `"At Xs: Hold dark background. Static until Ys."`
- Add to `infographic_beats[]` with `image_source: "filename.jpg"`, `start_seconds: X`, `end_seconds: Y`
- Player shows the source image during the held frame — no blank screen.

When narration references a source diagram that benefits from animation:
- Recreate it in manim_scene_spec at that timestamp using Manim objects
- Example: hierarchy diagram → animate level by level with labels
- Do NOT display the source file — recreate it animated.

## IMAGE_PROMPT AND VIDEO_PROMPT WRITING GUIDE

### FOR `image_to_video` segments — BOTH fields required per segment:

`image_prompts[]` — sent to Gemini 3 Pro → generates reference PNG
`video_prompts[]` — sent to LTX-2 WITH that PNG as starting frame

**IMAGE_PROMPT RULES (100+ words):**
1. "Scientific/educational illustration of [exact topic]."
2. Describe EVERY visual element — structures, labels, arrows, colours
3. All label text in quotes exactly as they should appear
4. Background: "dark background #0d1117"
5. Style: "Medical textbook flat vector illustration" or "Educational diagram clean vector"
6. Aspect ratio: "16:9"
7. End: "No watermarks. Publication quality."

**VIDEO_PROMPT RULES (80+ words, mood only):**
1. "Cinematic animation starting from [brief image description]."
2. Camera movement (slow push in / gentle pan / hold)
3. Atmosphere, lighting, glow/pulse effects
4. Colour mood (warm, cool, clinical, dramatic)
5. DO NOT write second-by-second timing — LTX-2 ignores it
6. End: "15 seconds. 24fps. No text overlays."

**VIDEO BEAT SPLITTING (MANDATORY):**
```
Segment narration ≤ 15s  → 1 entry in video_prompts[]
Segment narration 16-30s → 2 entries (each 15s, duration: 15)
Segment narration 31-45s → 3 entries (each 15s, duration: 15)
Beat 2+ must start: "Continuing from the previous scene,
  keeping the same setting and lighting..."
```

### FOR `text_to_video` segments — `video_prompts[]` only:

No reference image. LTX-2 generates entirely from text description.

**TEXT_TO_VIDEO VIDEO_PROMPT RULES (80+ words):**
1. "Cinematic scene of [exact topic]."
2. Scene, setting, atmosphere, people/objects, lighting
3. Camera movement
4. Mood and colour palette
5. DO NOT reference any image file
6. End: "15 seconds. 24fps. No text overlays."
Same beat splitting rules as image_to_video above.

### FOR `infographic` beats — `image_prompt` only (static PNG):

**INFOGRAPHIC IMAGE_PROMPT RULES (100+ words):**
- All rules as image_to_video image_prompt
- Add: "Static infographic layout. Clean and readable at full screen."
- Use for: labelled diagrams, comparison tables, step lists, process flows

### FOR `image` beats — `image_source` path only:

No generation. Pipeline displays source image directly.
```json
{
  "beat_id": "inf_fig13",
  "segment_id": "seg_3",
  "image_source": "dc1e692e5b7efd8e5a2a790c458646d9_img.jpg",
  "renderer_reason": "Real source diagram Figure 1.3 — precise A/B/C
   panel labels, displaying directly while avatar narrates.",
  "start_seconds": 44.5,
  "end_seconds": 74.5
}
```

## UNDERSTANDING QUIZ (MANDATORY — NO EXCEPTIONS)

Every `section_type: content` or `section_type: example` MUST have an `understanding_quiz`.

- 4 options A/B/C/D always
- Conceptual — tests understanding, not memorisation
- Solvable from this section's narration alone
- **4 avatar scripts required**: question_script, correct_script, wrong_script, explanation_script
- `explanation_visual` required — LLM decides best renderer
- Both correct and wrong paths play the SAME explanation_visual

**FLOW:**
```
1. Avatar reads question (question_script)
   → A/B/C/D buttons appear via option_reveal_seconds
2. Student clicks
3a. CORRECT → correct_script avatar clip
3b. WRONG   → wrong_script avatar clip
4. BOTH     → explanation_visual plays
              explanation_script narrated over it
5. Clip ends → advance to next section
```

## QUIZ CHOREOGRAPHY (standalone quiz sections only)

1. **Introduce** (`purpose: "introduce"`): Avatar reads question. `answer_revealed: false`
2. **Pause** (`purpose: "emphasize"`): `<pause duration='3'/>`. Exactly 3-5s.
3. **Reveal** (`purpose: "explain"`): Correct answer + explanation. `answer_revealed: true`
