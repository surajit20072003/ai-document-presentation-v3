## VISUAL GUIDELINES (MANDATORY) "TEACH AND SHOW"

### 1. UNIQUE PROMPTS PER SHOW SEGMENT (MANDATORY)
You MUST generate ONE entry in `segment_specs` for EACH SHOW segment (visual_layer = "show").
- **DO NOT** reuse the same prompt for multiple segments.
- **DO NOT** generate a single prompt for the entire section.
- **COUNT** your SHOW segments (e.g., seg_2, seg_4) and match the count in `segment_specs`.

### 2. WAN VIDEO LIMITS & BEATS (15s Max)
WAN 2.6 API produces 15s videos maximum.
- If a segment duration is **16-30s**: Provide 2 `beats` in that `segment_spec`.
- If a segment duration is **31-45s**: Provide 3 `beats`.
- For Beats 2+: Use prefix "Keeping the previous scene exactly the same, continue showing..."
- Each beat prompt MUST be 80+ words.

### 3. RENDERER SELECTION
1. **MATH / PHYSICS** -> **MANIM** (`renderer: "manim"`)
   - 1 `manim_scene_spec` per segment (NO 15s limit for Manim).

   - Uses Kie.ai WAN API (external) — better for organic/anatomical visuals.
3. **ALL OTHER VIDEO CONTENT** (history, general scenes, objects, landscapes, abstract concepts) -> **VIDEO** (`renderer: "video"`, `use_local_gpu: true`)
   - Uses Local GPU server — faster and free for generic visual content.
4. **STANDARD** -> **NONE** (`renderer: "none"`)

**CRITICAL**: `use_local_gpu` MUST be set for every `renderer: "video"` section.
- `use_local_gpu: false` → Only for Biology/Anatomy/Human Organ content
- `use_local_gpu: true`  → For ALL other video sections (default)
