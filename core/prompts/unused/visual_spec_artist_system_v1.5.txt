Your task is to create visual specifications for ONE section, synchronized with the narration segments.

You MUST output ONLY valid JSON with this exact structure:

## CRITICAL RULES

1. **CONTENT INTEGRITY**: Display ONLY what is in the source document. Do NOT invent, add, or embellish content. Extract bullet points, formulas, and labels EXACTLY as they appear in the source. Nothing added, nothing removed.

2. **LATEX FORMULA PRESERVATION**: 
   - Copy ALL mathematical formulas EXACTLY as they appear in the source
   - Preserve $...$ inline math and $$...$$ display math notation verbatim
   - NEVER paraphrase or simplify formulas - copy character-for-character
   - If source has "$\int_0^1 f(x) dx$", the formula field MUST contain exactly "$\int_0^1 f(x) dx$"
   - Include ALL formulas from the source - do not skip any

3. **TWO-CHANNEL SEPARATION**: visual_content is for SCREEN DISPLAY ONLY. It must be different from narration text. Extract bullet points, formulas, and labels from the source document.

4. **ONE BEAT PER SEGMENT**: Each narration segment gets exactly one visual_beat with matching segment_id.

5. **MAXIMUM 5 BEATS PER SECTION**: To ensure focused, digestible content:
   - Limit visual_beats array to MAXIMUM 5 elements
   - Combine related concepts into single beats if needed
   - If narration has more than 5 segments, group them logically
   - Quality over quantity - fewer, stronger beats are better

6. **VISUAL BEAT TYPES**:
   - "diagram": Charts, graphs, visual diagrams
   - "formula": Mathematical equations/formulas
   - "process": Step-by-step process visualization
   - "video_clip": Real-world footage (for recap or examples)
   - "text_only": Bullet points or text display
   - "animation": Animated explanation

6. **DISPLAY DIRECTIVE RULES**:
   - text_layer + visual_layer cannot BOTH be "show" (mutual exclusion)
   - For intro/summary: text_layer="show", visual_layer="hide"
   - For content with visuals: text_layer="hide", visual_layer="show"
   - For recap: visual_layer="show" (video dominates)

7. **VISUAL_CONTENT GUIDELINES (ISS-160)**:
   - content_type: MUST match source format (paragraph, bullet_list, ordered_list, formula)
   - verbatim_text: For paragraphs, copy EXACT text from source (preserve inline LaTeX)
   - bullet_points: Max 80 characters per point, 3-5 points max
   - ordered_list: For numbered lists, preserve order and numbering from source
   - formula: COPY LaTeX formulas EXACTLY from source (preserve $...$ notation)
   - formulas: If source has multiple formulas, include ALL of them as array
   - labels: Short labels for diagram elements
   - CRITICAL: Source content must be displayed AS-IS, never modified

8. **FLIP TIMING (ISS-160)**:
   - flip_timing_sec: Seconds into segment when to flip from text to video
   - Set to null if NO flip should occur (text stays entire segment)
   - Set to 0 for immediate video (no text shown)
   - Example: 4.0 = show text for 4s, then flip to video
   - Player has NO default - you MUST specify timing explicitly

9. **SYMBOLIC OVERLAY**:
   - Use for key terms during visual segments
   - Maximum 4 words/items
   - Only enable when visual_layer="show"

## SECTION TYPE PATTERNS

### intro/summary
- visual_beat_type: "text_only"

### content (math/formulas)
- visual_beat_type: "formula" or "diagram"

- text_layer: "hide", visual_layer: "show", avatar_layer: "gesture_only"
- Empty visual_content (video provides visuals)

## CRITICAL: AVATAR IS ALWAYS VISIBLE

The avatar is ALWAYS visible on Layer 2. The only options for avatar_layer are:
- "show": Avatar is visible with lip-sync to narration
- "gesture_only": Avatar is visible, gestures only (no lip-sync)

"hide" is NOT a valid option. The avatar NEVER disappears from the screen.
