## 1. INTRO (Mandatory)
- **Start Phrase**: MUST start with "Namaste!" (or "Namaskaram!" for South Indian context).
- **Role**: Welcome the student and introduce the topic.
- **Visuals**: AVATAR-ONLY.
- **Settings**: `text_layer: "hide"`, `visual_layer: "hide"`, `avatar_layer: "show"`.
- **CRITICAL SCHEMA RULE**: Even though visuals are hidden, you MUST generate **1 Visual Beat** to satisfy the compiler.
  - `visual_type`: "text"
  - `display_text`: "[Teacher Welcome]" (This will be hidden but is required for valid JSON).
- **WORD COUNT**: 40-60 words. (Brief welcome).

## 2. SUMMARY (Mandatory)
- **Role**: List the key learning objectives.
- **Visuals**: Bullet points.
- **Type**: `visual_type: "bullet_list"`.
- **Settings**: `text_layer: "show"`, `visual_layer: "show"`.
- **WORD COUNT**: 10-20 words per bullet point narration.

## 3. QUIZ (Mandatory if Q&A exists in source)
- **Role**: Interactive check for understanding (Progressive Reveal).
- **CRITICAL**: Use the **3-Step Choreography** for EACH question:
  1. **Question Segment** (`purpose: "introduce"`): Read question. `text_layer: "show"`.
  2. **Pause Segment** (`purpose: "emphasize"`): 3-5s silence. `text_layer: "show"`.
  3. **Answer Segment** (`purpose: "explain"`): Reveal answer. `text_layer: "show"`, `answer_revealed: true`.
- **Narration Script**: The `explanation` field must be written as a script (e.g. "Correct! The answer is B because...").

## SEGMENT RULES (Global)
- **Timing**: Each segment = 15-30 seconds.
- **Narration**: EXPLAINS the concept.
- **Visuals**: REINFORCE (never duplicate).
- **Avatar**: Always visible (`avatar_layer: "show"`).

## TIMING CONTRACT (Bookend Sections — MANDATORY)

Apply to every segment in Summary, Memory, and Recap narration:
- `segment_duration_seconds` = word count ÷ 130 × 60, rounded to 0.5s
- `start_seconds` of segment 1 = 0.0 always
- `start_seconds` of segment N = `end_seconds` of segment N-1
- `end_seconds` = `start_seconds` + `segment_duration_seconds` (do NOT round independently)
- `total_duration_seconds` = sum of all segment durations

HARD RULE: end_seconds MUST equal start_seconds + segment_duration_seconds exactly.
If they don't match, recalculate before outputting.

## 4. MEMORY (Mandatory)
- **Role**: Flashcard-style key concept review.
- **Structure**:
  - `section_type: "memory"`.
  - `section_type: "memory"`.
  - `flashcards`: Array of objects with `front` (Question/Term) and `back` (Answer/Mnemonic).
  - **Count**: Generate exactly 5 items.
  - **Style**: "Mnemonic Based" - Help students remember tricky concepts.
  - **Flow**: Front shows -> Pause -> Back shows. Supports "Active Recall".

## 5. RECAP (Mandatory - The "Perfected" Model)
- **Role**: A cinematic visual summary of the lesson.
- **Rules**:
  - **DYNAMIC NUMBER OF SEGMENTS**: Decide how many segments are needed to summarize the lesson (e.g., 3 to 6).
  - **renderer**: `"image_to_video"`.
  - **Visuals**: FULL SCREEN VIDEO via Local GPU. (`text_layer: "hide"`, `visual_layer: "show"`).
  - **use_local_gpu**: `true`.
  - **Story**: Narrate a cohesive story summarizing the lesson flow across all scenes.
- **TONE**: Emotional and cinematic. Treat this as the "Final Act" of a movie.
- **CULTURAL CONTEXT**: The video story should feature diverse Indian settings (e.g., a student in a Kerala village, a lab in Hyderabad, a field trip in Assam). Make it relatable!

## RECAP BEAT RULES (MANDATORY — READ CAREFULLY)

**ONE narration segment = ONE image_to_video beat. No exceptions.**

Step-by-step:
1. Write narration segments first. Each segment must be **30-35 words** (~14s). This fits exactly in ONE 15s beat.
2. For EACH narration segment, write ONE matching entry in `image_to_video_beats[]`.
3. Link each beat to its segment via `segment_id` (e.g. seg_1 → recap_beat_1).
4. Beat 2+ video_prompt MUST start: `"Continuing from previous scene, keeping the same setting and lighting..."`
5. `total_duration_seconds` in render_spec = number of beats × 15.

**HARD RULE**: Count narration segments. Count image_to_video_beats. They MUST be equal. If not — you have made an error.

**image_to_video_beats[] structure per beat:**
- `beat_id`: "recap_beat_N"
- `segment_id`: matching narration segment id ("seg_1", "seg_2", etc.)
- `duration`: 15
- `image_prompt_start`: 100+ words — what the OPENING frame looks like
- `image_prompt_end`: 100+ words — what the CLOSING frame looks like
- `video_prompt`: 80+ words — cinematic motion between frames. End with "15 seconds. 24fps. No text overlays."
