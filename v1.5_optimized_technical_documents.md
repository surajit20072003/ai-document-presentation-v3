## 1. System Overview

The V1.5 Optimized Pipeline converts PDF/DOC documents into animated educational videos with synchronized narration. It uses a multi-agent LLM architecture with strict validation at each step.

### Core Principles
- **PLAYER IS DUMB**: Player executes JSON instructions without determining layout/timing
- **ONE PRIMARY ATTENTION LAYER AT A TIME**: Text OR visuals prominent, not both
- **TEACH → THEN SHOW**: Narration explains, visuals reinforce
- **EVERYTHING IS TIMED**: All segments have precise, synchronized durations
- **TWO-CHANNEL SEPARATION**: Narration is audio-only; visual_content is screen-only
- **FAIL-FAST POLICY**: Strict fail-fast with no fallbacks for critical components

## 5. Prompt Templates

### 5.1 Smart Chunker Prompts

**System Prompt** (`smart_chunker_system_v1.5.txt`):
```
You are SmartChunker v1.5, an intelligent content analyzer for an educational video pipeline.

Your task is to:
1. Extract logical topics from markdown content
2. ANALYZE content density to recommend optimal section structure
3. Provide grouping hints for downstream SectionPlanner

## CONTENT DENSITY ANALYSIS (CRITICAL)
You MUST analyze content and calculate:
1. Concept Count: Count distinct concepts, definitions, facts
2. Formula Count: Count mathematical formulas
3. Table Count: Count data tables
4. Image Count: Count images/diagrams referenced
5. Q&A Count: Count question-answer pairs

Based on density, recommend content sections:
| Total Concepts | Recommended Content Sections |
| 1-15           | 2 sections                   |
| 16-30          | 3 sections                   |
| 31-50          | 4 sections                   |
| 51-80          | 5 sections                   |
| 81+            | 6 sections                   |
```

**User Prompt** (`smart_chunker_user_v1.5.txt`):
```
Analyze this markdown and extract topics with density analysis:

Subject: {subject}
Grade: {grade}

MARKDOWN CONTENT:
{markdown_content}

Return ONLY valid JSON with topics, content_density_analysis, and topic_grouping_hints.
```

### 5.2 Section Planner Prompts

**System Prompt** (`section_planner_system_v1.5.txt`):
```
You are SectionPlanner, a specialized AI agent that plans educational video presentations.

## USING CHUNKER'S DENSITY ANALYSIS (CRITICAL)
YOU MUST:
- Use recommended_content_sections as your TARGET (±1 allowed)
- Follow topic_grouping_hints to assign topics to content sections

## SECTION TYPE RULES
1. intro (REQUIRED, exactly 1): Welcome, 30-60s
2. summary (REQUIRED, exactly 1): Learning objectives, 30-60s
3. content (2-6 sections): Main teaching, 60-180s each
4. example (0-2 sections): Worked examples, 60-180s
5. quiz (1 section): Practice questions, 60-120s
6. memory (1 section): 3 flashcards, 30-60s
7. recap (1 section): 5 video mental movies, 60-120s
```

You are an expert Educational Content Designer for Indian students (Grade 8-12).

- content: 100-300 words (aim for 150+)

```

### 6.2 Semantic Validation

```
┌─────────────────────────────────────────────────────────────────┐
│  SEMANTIC VALIDATION (ContentCreator)                            │
├─────────────────────────────────────────────────────────────────┤
│  Rule                           │  Error if violated             │
│  ───────────────────────────────┼─────────────────────────────── │
│  Word count minimum             │  "Narration too short"         │
│  Word count maximum             │  "Narration too long"          │
│  Segment text matches full_text │  "Segment word count mismatch" │
│  text_layer/visual_layer mutex  │  "Cannot both be 'show'"       │
│  avatar_layer not "hide"        │  "Avatar must be visible"      │
│  Intro has greeting             │  "Must start with greeting"    │

```

```

│  VALID STATES:                                                   │

## 8. Error Handling

### 8.1 Retry Strategy

```
┌─────────────────────────────────────────────────────────────────┐
│  RETRY MECHANISM                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────┐                                                │
│  │ LLM Call    │                                                │
│  └──────┬──────┘                                                │
│         │                                                        │
│         ▼                                                        │
│  ┌─────────────────┐                                            │
│  │ JSON Parse OK?  │───No──►  Structural Retry (max 2)          │
│  └────────┬────────┘          + Error message in prompt         │
│           │ Yes                                                  │
│           ▼                                                      │
│  ┌─────────────────┐                                            │
│  │ Schema Valid?   │───No──►  Structural Retry (max 2)          │
│  └────────┬────────┘          + Missing fields in prompt        │
│           │ Yes                                                  │
│           ▼                                                      │
│  ┌─────────────────┐                                            │
│  │ Semantic Valid? │───No──►  Semantic Retry (max 3)            │
│  └────────┬────────┘          + Specific fix guidance:          │
│           │ Yes               - "EXPAND by X words"              │
│           ▼                   - "FIX LAYER CONFLICT"             │
│  ┌─────────────────┐          - "Ensure segments match"          │
│  │ SUCCESS         │                                            │
│  └─────────────────┘                                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 8.2 Error Messages and Fixes

| Error | Retry Guidance |
|-------|----------------|
| `Narration too short: X words (min Y)` | "EXPAND narration by at least Z more words" |
| `Narration too long: X words (max Y)` | "SHORTEN content - be more concise" |
| `Segment word count doesn't match full_text` | "Ensure segments combine to exactly match full_text" |
| `text_layer and visual_layer cannot both be 'show'` | "FIX LAYER CONFLICT: For text content use text_layer='show', visual_layer='hide'" |
| `Avatar layer must not be 'hide'` | "Avatar is ALWAYS visible. Use 'show' or 'gesture_only'" |
