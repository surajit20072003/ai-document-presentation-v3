# V2.5 Pipeline Routing Fix - Final Report

## Root Cause
**TWO hardcoded default parameter values** in function signatures were overriding the correct `pipeline_version` value from the dashboard:

### Bug Locations (FIXED)
1. **Line 1071**: `process_document_job_v15_v2(..., pipeline_version: str = "v15_v2")`
2. **Line 1187**: `process_markdown_job_v15_v2(..., pipeline_version: str = "v15_v2")`

### Why This Happened
When you upload a PDF:
- Dashboard sends: `'v15_v2_director'` ✅
- `app.py` receives: `'v15_v2_director'` ✅
- `process_document_job_v15_v2` is called
- **BUT** Python's default parameter mechanism was applying the hardcoded `"v15_v2"` value
- Result: Legacy pipeline executed instead of Director Mode

## Fix Applied
Changed BOTH function signatures to:
```python
pipeline_version: str = "v15_v2_director"
```

## Verification
Server restarted with clean code. Next job submission should show:
```
[ROUTING DEBUG] Received pipeline_version from form: 'v15_v2_director'
[PIPELINE DEBUG] ✓ BRANCH: Director Mode (V2.5)
```

## Impact
- ✅ PDF uploads now use V2.5 Director Mode
- ✅ Markdown uploads now use V2.5 Director Mode  
- ✅ 100% content fidelity via pointers
- ✅ Correct audio wiring
