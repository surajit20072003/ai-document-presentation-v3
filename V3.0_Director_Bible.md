**Purpose:** Single authoritative reference for anyone (agent or human) writing or debugging V3 Director outputs.

## 1. Philosophy

| Principle | Rule |
|---|---|
| **V3 Core Goal** | Next-level visual learning — Manim animations and cinematic videos illustrate topics in perfect sync with avatar narration. When narrator says "look at this", the visual appears at that exact moment. |
| **Visual-first** | Students see before they read. Manim / video / infographic replaces all text slides. |
| **Avatar = master clock** | `avatar.onended` is the only scene advance trigger. Player has zero timing logic. |
| **LLM owns sync** | Director writes narration duration AND visual duration in the same call. They always match exactly. |
| **Timing enforcer** | Phase 3.6 patches Manim durations post-avatar render to guarantee frame-perfect sync. |
| **Zero text on screen** | `text_layer: "hide"` always. No exceptions. |
| **Mandatory quiz** | Every content/example section outputs an `understanding_quiz`. No skip condition. |
| **Quiz is interactive HTML/JS** | Quiz and understanding sections use JS/HTML overlay in player for A/B/C/D interaction. Manim is NOT used for quiz display. |
| **No image sent to Director** | When source document has an image, pipeline sends a text `image_description` instead. Director uses this to write accurate `image_prompt`. |

## 2. Renderer Decision Tree

```
STEP 1 — Is it a molecule, formula, equation, or geometry?
  YES → renderer: "manim" — ALWAYS. No exceptions.
        Includes: benzene ring, NAPQI, drug structures, molecular bonds,
        sin/cos/tan, right triangles, geometric proofs, graphs/plots,
        force diagrams, circuit diagrams, chemical reaction pathways,
        metabolism pathway diagrams with labelled arrows

STEP 2 — Is it biology, anatomy, pharmacology in the body,
          farming/agriculture, history, geography, or real-world narrative?
  YES → renderer: "image_to_video" OR renderer: "text_to_video" — ALWAYS.
        image_to_video: when a reference image improves accuracy
          (anatomy, drug mechanisms, clinical scenarios, labelled diagrams)
        text_to_video: when a pure scene description is sufficient
          (history, geography, general narrative, farming, landscapes)

STEP 3 — Is it a conceptual example with no formula?
  (e.g. "measuring height of a street light", "cricket pitch length",
   "how a doctor takes blood pressure")
  → LLM DECIDES based on complexity:
      simple  → renderer: "manim"       (infographic: shapes + labels + arrows)
      complex → renderer: "image_to_video" or "text_to_video" (cinematic)

STEP 4 — Is it a static labelled diagram?
  → renderer: "infographic"
    Gemini 3 Pro generates PNG → displayed as static image in player
    Use when: comparison tables, step lists, quick reference diagrams,
    simple labelled illustrations where motion adds no value

STEP 5 — Is it a source image to display as-is while avatar narrates?
  → renderer: "image"
    Source document image displayed directly in player
    No generation needed

STEP 6 — Is it intro / summary / memory / recap?
  YES → renderer: "none" (avatar only)
```

## 3. Renderer Reference Table

| Renderer | What it produces | Use when |
|---|---|---|
| `manim` | Manim Python → MP4 (server-rendered) | Molecules, formulas, geometry, graphs, STEM diagrams |
| `image_to_video` | Gemini 3 Pro PNG → LTX-2 MP4 | Biology, anatomy, clinical, conceptual complex scenes |
| `text_to_video` | Text prompt → LTX-2 MP4 | History, geography, farming, narrative — no reference image needed |
| `infographic` | Gemini 3 Pro PNG → static display | Comparison tables, step lists, simple labelled diagrams |
| `image` | Source document image → static display | Display source image while avatar narrates |
| `none` | Avatar only | Intro, summary, memory, recap |

## 4. Timing Contract (MANDATORY)

The Director is the single source of truth for all timing.

```
Rules:
- segment_duration_seconds = narration word count ÷ 130 words/min × 60
  Round to nearest 0.5s
- start_seconds of first segment = 0.0 always
- start_seconds of segment N = end_seconds of segment N-1
- end_seconds = start_seconds + segment_duration_seconds
- total_duration_seconds = sum of all segment durations
- Manim animation MUST fill exactly segment_duration_seconds
  → state this explicitly at end of manim_scene_spec: "Duration exactly Xs."
- Avatar TTS is written to match segment_duration_seconds
- image_to_video and text_to_video → LTX-2 always generates 15.0s
  → write narration for these segments to be exactly 15.0s (~32-33 words)
- infographic and image → no fixed duration; match narration duration
```

## 5. Image and Video Prompt Writing Guide

### When to generate image_prompt and video_prompt

| Renderer | image_prompt | video_prompt |
|---|---|---|
| `image_to_video` | ✅ Required (100+ words) | ✅ Required (80+ words) |
| `text_to_video` | ❌ Not needed | ✅ Required (80+ words) |
| `infographic` | ✅ Required (100+ words) | ❌ Not needed |
| `image` | ❌ Not needed | ❌ Not needed |
| `manim` | ❌ Not needed | ❌ Not needed |

**The same image_prompt is used twice:**
1. Sent to Gemini 3 Pro → generates reference PNG
2. That PNG sent to LTX-2 as starting frame (image_to_video only)

### image_prompt Rules (100+ words)

1. Start: "Scientific/educational illustration of [exact topic]."
2. Describe EVERY visual element — structures, labels, arrows, colours
3. Specify ALL label text in quotes exactly as they should appear on screen
4. Specify background: "dark background #0d1117"
5. Specify colour coding: what colour means what (e.g. green=safe, red=toxic)
6. Specify style: "Medical textbook flat vector illustration" OR "Cinematic photorealistic" OR "Educational diagram clean vector"
7. Specify aspect ratio: "16:9"
8. If image_description provided from source — describe it accurately, correct any scientific errors
9. End: "No watermarks. No copyright symbols. Publication quality."

**GOOD IMAGE PROMPT EXAMPLE (liver metabolism):**
```
Scientific medical illustration of paracetamol hepatic metabolism
inside a human liver hepatocyte cell. Dark background #0d1117.

TOP CENTRE: Paracetamol molecular structure — benzene ring with
HO group left, NH-CO-CH3 acetamide group right. Label "PARACETAMOL"
white bold beneath.

LEFT PATHWAY green arrows: labeled "90% glucuronidation" →
enzyme box "UDP-GLUCURONOSYLTRANSFERASE (UGT)" in green →
product box "PARACETAMOL GLUCURONIDE — Non-toxic water-soluble
metabolite" → arrow to label "Renal excretion".

RIGHT PATHWAY red arrows: labeled "10% CYP450 → NAPQI" →
enzyme "CYP450 (mostly CYP2E1)" → NAPQI quinone-imine structure
labeled "NAPQI" in red → two branches: (1) "Glutathione (GSH)
conjugation" via "GLUTATHIONE S-TRANSFERASE (GST)" → "NAPQI-
GLUTATHIONE CONJUGATE" white box; (2) "Binding to hepatocellular
proteins" → "Hepatic cell damage" red.

CENTRE: Endoplasmic reticulum layered grey-purple membranes,
purple nucleus, label "Human hepatocyte". Brown organic cell
boundary. Label "Cytoplasm" bottom left.

Medical textbook flat vector illustration. Green=safe, red=toxic.
White labels on dark background. 16:9. No watermarks.
```

### video_prompt Rules (80+ words — mood only)

1. Start: "Cinematic animation starting from [brief description of image]." (image_to_video)
   OR: "Cinematic video of [scene description]." (text_to_video)
2. Describe camera movement (slow push in / gentle pan / orbit / hold)
3. Describe atmosphere: lighting, tone, mood
4. Describe glow/pulse/particle effects on key elements
5. Describe colour mood (warm, clinical, dramatic, documentary)
6. **NEVER write second-by-second choreography** — LTX-2 ignores timing instructions
7. End: "15 seconds. 24fps. No text overlays."

**CRITICAL LTX-2 RULE:**
- ✅ DO write: mood, atmosphere, colours, camera movement, lighting, particle effects
- ❌ DO NOT write: "at 3s the arrow lights up", "at 6s the enzyme activates"
- LTX-2 reads mood only — choreography instructions are completely ignored

**GOOD VIDEO PROMPT EXAMPLE:**
```
Cinematic animation starting from a detailed medical diagram of
paracetamol metabolism inside a human liver cell on dark background.

Camera slowly pushes in over 15 seconds. Soft bioluminescent glow
pulses along the green left pathway — warm, safe, reassuring light.
The red right pathway glows with a clinical warning pulse — sharp,
urgent red around the NAPQI structure. Endoplasmic reticulum
membranes ripple softly. Purple nucleus glows gently. Cell boundary
shimmers with warm amber light.

Professional medical documentary atmosphere. Smooth fluid motion.
Vibrant green and red pathway colours. Subtle particle drift.
Educational tone. 15 seconds. 24fps. No text overlays.
```

## 6. Manim Scene Spec Writing Guide

Every `renderer: "manim"` segment_spec MUST have a `manim_scene_spec` of 100+ words.

### Manim colour names to use
`GOLD, TEAL, RED, BLUE, WHITE, YELLOW, GREEN, PURPLE`

### 8-item checklist for every manim_scene_spec

1. Background colour (always dark: `background_color=BLACK` or `#0d1117`)
2. What objects appear at t=0
3. Animation sequence — explicit second-level timing for EVERY object
4. Manim animation methods (`Create, Write, FadeIn, GrowFromCenter, Transform, Indicate`)
5. Colours for every object using Manim colour names
6. LaTeX formulas written in LaTeX syntax e.g. `MathTex(r"\sin\theta = \frac{O}{H}")`
7. Exact label text in quotes
8. Final freeze frame description + "Duration exactly Xs." + `frame_width=N`

**GOOD EXAMPLE:**
```
Dark background. frame_width=10. Duration exactly 18s.
At 0s: FadeIn dark background over 0.5s.
At 1s: Create right triangle ABC using Polygon in TEAL over 2s.
  Vertices: A bottom-left [−3,−2], B bottom-right [3,−2], C top-right [3,2].
At 3s: Write labels "A", "B", "C" at each vertex in BLUE, 1s.
At 4s: Create side BC highlighted in RED over 1.5s.
  Write label "Opposite" next to BC in RED, 0.8s.
At 6s: Create hypotenuse AC in GOLD using GrowFromCenter over 2s.
  Write label "Hypotenuse" next to AC in GOLD, 0.8s.
At 9s: Create right-angle square at B in WHITE over 0.5s.
At 10s: Write MathTex(r"\sin\theta = \frac{BC}{AC}") in GOLD
  at screen centre using Write() over 2s.
At 13s: Indicate(triangle, color=GOLD) once — pulse.
At 15s: Final static frame holds for 3s.
Complexity: medium.
```

**BAD EXAMPLE (rejected — vague):**
```
Show a right triangle with sine labels and a formula.
```

### Additional Manim rules
- Always end with freeze frame (last 2–3s static)
- Molecules: use `Polygon`, `Circle`, `Line`, `Text` — NOT external images
- Graphs: use `Axes` with labelled axes and `plot()`
- Pathways: use `Arrow`, `Rectangle`, `Text` boxes with connecting arrows
- Always specify `frame_width` (default 10, increase to 14 for wide diagrams)
- Last line of spec MUST be: `"Duration exactly Xs."`

## 7. Source Image Handling

When `image_description` is provided in the prompt context:

```
This is a TEXT DESCRIPTION of an image found in the source document.
The actual image file is NOT sent — pipeline sends text description only.

Director must:
1. Use the description to write a more accurate image_prompt
2. Set image_source to the provided file path
3. Set image_mode: "video_reference"
4. Correct any scientific errors identified from the description
5. The pipeline will use the source image as LTX-2 starting frame
```

### image_mode values

| image_mode | When to use | renderer |
|---|---|---|
| `video_reference` | Source image is a photo or realistic illustration | `image_to_video` |
| `texture` | Source image is a diagram/chart to annotate | `manim` (load as ImageMobject background) |
| `interactive_hotspot` | Source image is a labelled diagram with parts to explain individually | `manim` (load as ImageMobject + highlight overlays) |

## 8. Understanding Quiz — Mandatory, No Exceptions

Every `section_type: content` or `section_type: example` section MUST output `understanding_quiz`.

```json
"understanding_quiz": {
  "question": "Question text?",
  "options": { "A": "...", "B": "...", "C": "...", "D": "..." },
  "correct": "B",
  "explanation": "One sentence explaining why B is correct.",
  "narration": {
    "question_script": "Avatar reads full question + all 4 options aloud.",
    "correct_script": "Avatar reacts positively and explains why correct.",
    "wrong_script": "Avatar says it is okay and explains correct answer.",
    "option_reveal_seconds": [3.5, 5.8, 8.1, 10.4]
  }
}
```

**Rules:**
- Always 4 options A/B/C/D — hard fail if fewer
- All 3 narration scripts required — used to generate 3 avatar clips
- `option_reveal_seconds` — 4 floats, one per option. Second within `question_script` clip when each option button visually appears. Estimate at ~130 wpm.

## 9. Quiz Section — Full Schema with Explanation Visual

### Architecture

Quiz sections are **interactive HTML/JS** in the player — NOT Manim.
The player renders A/B/C/D buttons. Student clicks. Player handles correct/wrong flow.
Manim, infographic, and video are used ONLY for the explanation visual after the answer.

### Quiz Flow (Player)

```
1. Player renders A/B/C/D question card (HTML/JS)
2. Avatar reads question aloud
3. Options appear one-by-one per option_reveal_seconds
4. Student clicks answer
   IF correct → avatar: "Well done! Here is why..." → explanation_visual plays
   IF wrong   → avatar: "That is okay, let me explain..." → explanation_visual plays
5. SAME explanation_visual plays for both correct and wrong
6. Avatar explanation ends → advance to next question or section
```

### Full Quiz Section Schema

```json
{
  "section_type": "quiz",
  "quiz_origin": "understanding",
  "questions": [
    {
      "question_id": "q1",
      "question_text": "What is the ratio of Opposite to Hypotenuse?",
      "options": { "A": "cosine", "B": "sine", "C": "tangent", "D": "secant" },
      "correct_option": "B",
      "explanation": "Sine = Opposite / Hypotenuse (SOH in SOH-CAH-TOA).",
      "avatar_clips": {
        "question":     "avatars/en/quiz_{id}_q1_question.mp4",
        "correct":      "avatars/en/quiz_{id}_q1_correct.mp4",
        "wrong":        "avatars/en/quiz_{id}_q1_wrong.mp4",
        "explanation":  "avatars/en/quiz_{id}_q1_explanation.mp4"
      },
      "narration": {
        "question_script":    "Here is your question: What is the ratio of Opposite to Hypotenuse? Is it A: cosine, B: sine, C: tangent, or D: secant?",
        "correct_script":     "Well done! That is correct. Let me show you exactly why.",
        "wrong_script":       "That is okay — everyone gets this. Let me explain the correct answer.",
        "explanation_script": "Sine is defined as Opposite divided by Hypotenuse. Look at this right triangle — the side opposite the angle divided by the longest side gives us sine. This is the S in SOH-CAH-TOA.",
        "option_reveal_seconds": [3.5, 5.5, 7.5, 9.5]
      },
      "explanation_visual": {
        "renderer": "manim",
        "duration_seconds": 15.0,
        "manim_scene_spec": "Dark background. frame_width=10. Duration exactly 15s. At 0s: Create right triangle in TEAL over 2s. At 2s: highlight Opposite side BC in RED, Write label Opposite. At 4s: highlight Hypotenuse AC in GOLD, Write label Hypotenuse. At 7s: Write MathTex(r'\\sin\\theta = \\frac{Opposite}{Hypotenuse}') in GOLD at centre over 2s. At 10s: Indicate formula with gold pulse. At 12s: freeze static frame 3s. Complexity simple.",
        "image_prompt": null,
        "video_prompt": null,
        "display_text": "Right triangle with Opposite and Hypotenuse highlighted, sine formula"
      }
    }
  ]
}
```

### explanation_visual Rules

| Field | Rule |
|---|---|
| `renderer` | Any of: `manim`, `image_to_video`, `text_to_video`, `infographic`, `image` |
| `duration_seconds` | Must match explanation_script narration duration |
| `manim_scene_spec` | Required if renderer is manim. 100+ words. Explicit timing. |
| `image_prompt` | Required if renderer is image_to_video or infographic |
| `video_prompt` | Required if renderer is image_to_video or text_to_video |
| Content | Must visually PROVE why the correct answer is correct |
| Same for all | Explanation visual plays regardless of correct or wrong — same asset |

## 13. Hard Rules (Non-Negotiable)

1. **LLM owns sync** — Director writes narration + visual duration together. Player never adjusts timing.
2. **Avatar is master clock** — `avatar.onended` is the only scene advance trigger.
3. **No text in content** — `text_layer: "hide"` always in content/example sections.
4. **Manim freeze on completion** — never loop, never clear; freeze last frame.
5. **4 avatar clips per quiz question** — question, correct, wrong, explanation. Always.
6. **Understanding quiz is mandatory** — every content/example section, no skip condition.
7. **All manim_scene_spec 100+ words** — shorter fails validator.
8. **image_prompt 100+ words** — shorter fails validator.
9. **video_prompt 80+ words** — shorter fails validator.
10. **Molecules always Manim** — no exceptions. Never use image_to_video for molecules or formulas.
11. **Quiz is HTML/JS** — A/B/C/D interaction is handled by player. Never use Manim for quiz cards.
12. **Same explanation_visual for correct and wrong** — one asset generated, played on both paths.
13. **No image sent to Director** — pipeline sends text `image_description` only. Director uses it to write `image_prompt`.
14. **LTX-2 ignores choreography** — video_prompt must be mood/atmosphere only. No timing instructions.
15. **start_seconds + end_seconds required** — every segment_spec must have cumulative timing.
16. **Timing contract enforced** — total_duration_seconds = sum of all segment_duration_seconds.

## 14. Sync Pipeline

```
Phase 1: Director     → writes narration + visual specs per section
Phase 2: Duration     → word-count math sets segment_duration_seconds
Phase 3: Manim        → Python renders MP4 to exact segment_duration_seconds
Phase 3: LTX-2        → always generates 15.0s MP4 (director writes narration to match)
Phase 4: Infographic  → Gemini 3 Pro generates PNG (no duration)
Phase 5: Avatar GPU   → renders MP4 with voice, writes avatar_duration_seconds
Phase 3.6: Enforcer   → patches Manim durations to match real avatar MP4 duration
Player:               → avatar.onended advances scene. No timing logic in player.
```

## 15. Validator Rules

| Rule | Condition |
|---|---|
| `text_layer_not_hidden` | `text_layer` ≠ `"hide"` in content/example — hard fail |
| `renderer_invalid` | renderer not in valid set — hard fail |
| `manim_spec_missing` | manim renderer + no manim_scene_spec — hard fail |
| `manim_spec_too_short` | manim_scene_spec < 100 words — hard fail |
| `image_prompt_missing` | image_to_video or infographic + no image_prompt — hard fail |
| `image_prompt_too_short` | image_prompt < 100 words — hard fail |
| `video_prompt_missing` | image_to_video or text_to_video + no video_prompt — hard fail |
| `video_prompt_too_short` | video_prompt < 80 words — hard fail |
| `timing_missing` | segment_spec missing start_seconds or end_seconds — hard fail |
| `timing_gap` | end_seconds of N ≠ start_seconds of N+1 — hard fail |
| `duration_mismatch` | total_duration ≠ sum of segment_durations — hard fail |
| `understanding_quiz_missing` | content/example without understanding_quiz — hard fail |
| `quiz_options_insufficient` | < 4 options — hard fail |
| `quiz_narration_missing` | any of 4 scripts missing — hard fail |
| `explanation_visual_missing` | quiz question without explanation_visual — hard fail |
| `image_mode_missing_source` | image_mode set + image_source null — hard fail |
| `narration_too_short` | content < 150 words, example < 100 words — hard fail |
