# Content Completeness Validator Implementation Plan

## Overview

Implement a **programmatic validation checkpoint** that runs **after Director LLM generation** but **before asset generation** (Manim/WAN/Avatar) to ensure `presentation.json` has captured 100% of content from the source markdown.

### Key Requirements (User Confirmed)

✅ **Validation Scope:**
- All images from source markdown are referenced in `presentation.json`
- All major topics/headings from source markdown are covered
- All key terms/concepts are present
- Total word count/content volume is preserved

✅ **Validation Strictness:** Retry with enhanced prompt telling LLM what it missed

✅ **Image Validation:** Ensure `.png` files saved match what's referenced in `presentation.json`

✅ **Topic Coverage:** Smart Chunker provides validation metadata

✅ **Pipeline Position:** After Director LLM, before asset generation

## Proposed Changes

### Core Validator Module

#### [NEW] [content_completeness_validator.py](file:///c:/Users/email/Downloads/AI-Document-presentation/ai-doc-presentation/core/validators/content_completeness_validator.py)

**Purpose:** Validate that `presentation.json` captures all source markdown content before expensive asset generation.

**Validation Checks:**

1. **Image Coverage Check**
   - Compare images in `images/` directory vs `image_id` references in `presentation.json`
   - Ensure all extracted `.png` files are referenced
   - Report: Missing images, orphaned references

2. **Topic Coverage Check**
   - Use Smart Chunker's `topics` metadata as ground truth
   - Match each chunker topic to sections in `presentation.json`
   - Ensure all `topic_id` values from chunker appear in final presentation
   - Report: Missing topics

3. **Key Terms Check**
   - Extract `key_terms` from Smart Chunker output
   - Verify these terms appear in narration segments
   - Report: Missing key terms

4. **Content Volume Check**
   - Compare word count: source markdown vs total narration text
   - Tolerance: ±20% (some expansion/contraction is normal)
   - Report: Significant volume mismatch

**Output Structure:**

```python
{
    "validation_status": "passed" | "failed",
    "checks": {
        "image_coverage": {
            "status": "passed" | "failed",
            "total_images": 5,
            "referenced_images": 5,
            "missing_images": [],
            "orphaned_references": []
        },
        "topic_coverage": {
            "status": "passed" | "failed",
            "total_topics": 3,
            "covered_topics": 3,
            "missing_topics": []
        },
        "key_terms": {
            "status": "passed" | "failed",
            "total_terms": 12,
            "found_terms": 12,
            "missing_terms": []
        },
        "content_volume": {
            "status": "passed" | "failed",
            "source_word_count": 1500,
            "presentation_word_count": 1450,
            "coverage_percent": 96.7
        }
    },
    "retry_prompt_enhancement": "MISSING CONTENT:\n- Image: diagram_1.png not referenced\n- Topic: 'Photosynthesis Process' not covered\n..."
}
```

**Change 2: Validation Checkpoint** (after line 164, before Manim codegen at line 254)

```python
# --- PHASE 2.7: CONTENT COMPLETENESS VALIDATION ---
if pipeline_version == "v15_v2_director" and not dry_run:
    log_status("validation", "Validating content completeness...")
    
    from core.validators.content_completeness_validator import ContentCompletenessValidator
    
    validator = ContentCompletenessValidator()
    validation_result = validator.validate(
        presentation=presentation,
        chunker_output_path=str(output_path / "artifacts" / "01_chunker.json"),
        images_dir=str(output_path / "images"),
        source_markdown=markdown_content
    )
    
    # Save validation report
    validation_path = output_path / "artifacts" / "completeness_validation.json"
    with open(validation_path, "w", encoding="utf-8") as f:
        json.dump(validation_result, f, indent=2)
    
    if validation_result["validation_status"] == "failed":
        log_status("validation", "Content validation FAILED - Missing content detected")
        
        # RETRY MECHANISM
        retry_prompt = validation_result.get("retry_prompt_enhancement", "")
        
        logger.warning(f"Validation failed. Retry prompt:\n{retry_prompt}")
        
        # Re-run Director with enhanced prompt
        director = PartitionDirectorGenerator(config=config)
        v2_output = director.generate_presentation_partitioned(
            markdown_content=markdown_content,
            subject=subject,
            grade=grade,
            images_list=images_list,
            update_status_callback=log_status,
            generation_scope=generation_scope,
            output_dir=str(output_dir),
            missing_content_hint=retry_prompt  # NEW PARAMETER
        )
        
        presentation = v2_output
        
        # Validate again (only once - then fail hard)
        validation_result_retry = validator.validate(...)
        
        if validation_result_retry["validation_status"] == "failed":
            raise PipelineUnifiedError(
                f"Content validation failed after retry:\n{validation_result_retry}",
                "content_validation"
            )
        
    log_status("validation", "Content validation PASSED ✓")
```
