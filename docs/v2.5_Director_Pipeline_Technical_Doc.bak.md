## 4. Phase-by-Phase Breakdown

### Phase 1: Image Processing
**File**: `core/image_processor.py`  
**Function**: `save_datalab_images()`, `extract_image_refs_from_markdown()`

```
Markdown Input  →  Extract Images  →  Save to /images/  →  images_dict
```

### Phase 2: LLM Content Generation
**File**: `core/unified_content_generator.py`  
**Function**: `generate_presentation()`, `transform_to_player_schema()`

```
markdown + subject + grade  →  OpenRouter API  →  V2 JSON  →  Player Schema
```

- Uses `DIRECTOR_MODEL` from `.env` (or default)
- Single LLM call generates ALL sections
- Output is transformed to v1.3 schema

### Phase 2b: Merge & Order
**File**: `core/merge_step_v15.py`  
**Function**: `merge_agent_outputs()`, `_order_sections()`

```python
# Critical: Case-insensitive sorting
```

### Phase 3: Manim Code Generation
**File**: `core/agents/manim_code_generator.py`  
**Function**: `ManimCodeGenerator.generate()`, `integrate_manim_code_into_section()`

**V2.5 Prompts**:
- System: `core/prompts/manim_system_prompt_v25.txt`
- User: `core/prompts/manim_user_prompt_v25.txt`

**Model**: Configured via `MANIM_MODEL` in `.env` (default: `anthropic/claude-sonnet-4`)

```
Section with renderer="manim"  →  LLM generates Python code  →  Stored in section
```

**Key V2.5 Rules**:
- ASCII-only text (no Unicode ₹, •, etc.)
- Strict timing sync with narration
- `FadeOut` before adding new objects

### Phase 4: Avatar Trigger (Fire-and-Forget)
**File**: `core/agents/avatar_generator.py`  
**Function**: `AvatarGenerator.submit_parallel_job()`

```
All sections  →  Submit to Avatar API (69.197.145.4:5001)  →  Background polling
```

### Phase 4b: TTS Generation
**File**: `core/tts_duration.py`  
**Function**: `update_durations_simplified()`

```
Segments  →  TTS Provider (our_tts/edge_tts/narakeet/estimate)  →  audio files + durations
```

**TTS Providers**:
- `our_tts`: Custom API at `http://69.197.145.4:8000` (timeout: 5s/15s)
- `edge_tts`: Free Microsoft Edge TTS
- `narakeet`: Paid API
- `estimate`: No audio, duration estimate only (Default for V2.5)

> **Note**: TTS is explicitly **DISABLED** for V2.5 Pipelines to optimize performance. It relies on `estimate` mode.

### Phase 5: Visual Rendering (Split Execution)
**File**: `core/renderer_executor.py`  
**Function**: `render_all_topics()`, `submit_wan_background_job()`

**Part A: Blocking Manim Phase (Synchronous)**
- **Action**: Filters for `renderer="manim"`.
- **Behavior**: BLOCKS the pipeline until all Manim videos are generated. 
- **Reason**: Math/Code logic must be verified before considering the job "done".

**Part B: Async WAN Phase (Background)**
- **Action**: Spawns a background thread (`submit_wan_background_job`).
- **Behavior**: Fire-and-Forget. The main pipeline proceeds immediately to completion.
- **Batching**: Processes WAN videos in **Batches of 5** to respect GPU parallelism limits.
- **Live Updates**: Updates `presentation.json` and `analytics.json` in real-time as batches complete.

```
Manim Topics  →  Render (Wait)  →  Job Continue
WAN Topics    →  Background Thread  →  Batch(5)  →  Update JSONs  →  Next Batch
```

**Critical Fix (2026-01-07)**: Results must be applied back to presentation:

## 6. Common Issues & Fixes

### Issue 1: Section Order Wrong
**Symptom**: Summary appears at position 11 instead of 2  
**Cause**: Case-sensitive lookup ("Summary" vs "summary")  
**File**: `core/merge_step_v15.py`  
**Fix**: `.lower()` on section_type before sorting

### Issue 2: Videos Generated But Not Playing
**Symptom**: Videos exist in `/videos/` but player shows blank  
**Cause**: Missing result application in `pipeline_unified.py`  
**Fix**: Add loop to apply `video_path`, `beat_videos`, `recap_video_paths` from render results

### Issue 3: Audio Fails, Player Freezes
**Symptom**: No time updates, bullets don't appear  
**Cause**: Missing audio = missing time source  
**Fix**: `BrowserTTSPlayer` fallback in `player_v2.js`

### Issue 4: TTS Timeouts
**Symptom**: Pipeline stalls for 60+ seconds per segment  
**Cause**: 60s default timeout on TTS API  
**Fix**: Reduced to 5s/15s tuple + fail-fast on 500 errors

## 7. Player Features (V2.5 Enhancements)

### 7.1 Progressive Playback (Silent Mode)
The player now supports immediate playback even if assets are missing:
1.  **Timer Fallback**: Uses WPM-based durations from `presentation.json` if audio/avatar is missing.
2.  **Visual Indicator**: Shows "🔇 Silent Mode" badge when running on timer.
3.  **Smart Switching**: Automatically switches to Avatar or Audio if they finish generating and you reload/switch slides.

### 7.2 Subtitles / Narration Overlay (Karaoke Style)
Added to support "Silent Mode" and accessibility:
- **Display**: Semi-transparent overlay with dimmed text.
- **Karaoke Effect**: Words light up white recursively as segment plays.
- **Calculated Sync**: Uses WPM-based progress interpolation.
- **Logic**: Updates automatically via `updateActiveSegment()` loop.
- **Visibility**: Auto-hides when no text is active.

### 9.1 Phase Retry System (`/job/<id>/retry_phase`)

The Phase Retry system allows you to regenerate specific phases without rerunning the entire pipeline.

**Request Body:**
```json
{
  "phase": "manim_codegen",    // Required: manim_codegen | video_render | tts_generation
  "section_ids": [2, 4, 6, 8]  // Optional: Force retry these specific section IDs
}
```

**Available Phases:**
| Phase | What it Does | When to Use |
|-------|--------------|-------------|
| `manim_codegen` | Regenerates Python Manim code from spec | LLM code truncation, syntax errors |
| `video_render` | Re-renders Manim/WAN videos | Rendering failures, missing videos |
| `tts_generation` | Re-generates TTS audio | Audio failures, duration mismatch |

**Retry Behavior:**

1. **Auto-Detection Mode** (no `section_ids` provided):
   - Only retries sections **without** existing code/video
   - Sections with valid code are **SKIPPED**
   - Safe to run multiple times

2. **Force Mode** (with `section_ids` provided):
   - **Forces regeneration** even if code/video exists
   - Use this to apply new prompts (e.g., V25 Manim prompts)
   - Example: `"section_ids": [2, 4, 6, 8]`

**Dashboard Usage:**
1. Click **"Retry Phase"** button on a completed job
2. Select phase: `Manim Code Generation`
3. **To force regeneration**: Enter section IDs: `2, 4, 6, 8`
4. Leave empty for auto-detection mode
5. Click **"Retry Phase"**

**Example Response:**
```json
{
  "status": "success",
  "phase": "manim_codegen",
  "result": {
    "success": [{"section_id": 2, "code_length": 5420}],
    "failed": [],
    "skipped": [{"section_id": 6, "reason": "Already has valid code"}]
  }
}
```
